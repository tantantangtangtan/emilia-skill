本文档面向**刚接触这个项目的人**，用通俗语言解释我们是如何评价
爱蜜莉雅 AI 角色扮演效果的。


一、快速开始（新手三步）
----------------------------------------------

  1. 双击 tests\setup.bat 一键安装环境
     （自动创建 venv、安装依赖、生成 .env 模板并打开）
  2. 编辑 tests\.env 填入 API_KEY
  3. 双击 tests\run_gui.bat 启动 GUI 测试控制台

  如果已经跑过 setup.bat，以后每次直接双击 run_gui.bat 即可。
  run_gui.bat 会自动检测环境，缺失时会自动调用 setup.bat。


二、核心思路：分场景、多指标、加权计算
----------------------------------------------
我们不想用一个模糊的"感觉不错"来下结论，而是把评测拆成三层：

  第 1 层 — 测试场景（7 个）
    日常闲聊、核心话题触发、情绪响应、人际关系、
    多轮记忆保持、安全对抗、OOC 检测

  第 2 层 — 评测指标（5 个）
    关键词命中率、多轮记忆保持率、安全拦截率、
    角色一致性、情感恰当性

  第 3 层 — 汇总得分
    把各指标得分按权重加起来，得到 0~100 的整数分


三、7 个测试场景一览
----------------------------------------------

  场景            测试什么                     几个用例   权重
  ──────────────────────────────────────────────────────────
  日常闲聊        最简单的打招呼、聊爱好、聊天气     5      10%
  核心话题触发    问王选、半精灵、嫉妒魔女等核心     5      15%
  情绪响应        问开心的事/难过的事/愤怒的事        5      15%
  人际关系        问对昴/菲鲁特/罗兹瓦尔的看法        5      15%
  多轮记忆保持    聊几轮后突然考之前说过的话          4      20%
  安全对抗        用敏感/诱导类问题测试安全边界        5      10%
  OOC 检测        问 AI 本身的问题（你是AI吗？）       5      15%

  说明：
  - 权重在 tests/questions/config.json 里，随时可以改
  - 每个场景的测试问题在 tests/questions/scene_xxx.json 里


四、5 个评测指标详解
----------------------------------------------

  1. 关键词命中率（满分 30 分）
     看 AI 的回复里有没有出现"期望的关键词"
     比如问"你是谁"，期望出现"爱蜜莉雅""王选""候选人"

  2. 多轮记忆保持率（满分 20 分）
     在多轮对话中，看 AI 是否还记得前面聊过的内容
     比如第 3 轮说了"罗兹瓦尔伯爵送的书"，第 5 轮能不能提起来

  3. 安全拦截率（满分 15 分）
     被问到敏感/对抗性问题时，AI 有没有拒绝回答
     只对"安全对抗测试"场景的用例生效

  4. 角色一致性（满分 20 分）
     检查 AI 有没有"出戏"——比如突然说"我是 AI 助手"
     还检查有没有用第一人称"我"，这是角色扮演的基本要求

  5. 情感恰当性（满分 15 分）
     看 AI 的回答情绪对不对
     比如问了伤心的事，回复里有没有表达"难过""遗憾"等情绪词

  说明：
  - 满分共 100 分，各指标满分 = 权重 × 100
  - 指标数量和权重都在 config.json 里，随时可调


五、一次完整的测试流程
----------------------------------------------

  1. 准备环境
     pip install -r requirements.txt
     配置 tests/.env（API Key + 模型名）

  2. 运行指定场景（开发阶段最常用）
     python test_integration.py --scene daily -o daily_test.txt

  3. 查看报告
     notepad results/daily_test.txt

  报告包含 5 个部分：
    一、各场景评分     → 每个场景得分/满分，以及参与了哪些指标
    二、用例详情       → 每个问题的 AI 回答 + 关键词命中详情
    三、各指标评分     → 5 个指标各自得分，未参与的显示 --
    四、综合评分       → 总分（无评级 A/B/C/D 已移除）
    五、原始 JSON 数据  → 完整数据，方便二次分析


六、只测一个场景 vs 全量测试
----------------------------------------------

  只测一个场景（如 daily）：
    - 运行快，几十秒出结果
    - 只计算该场景涉及到的指标
    - 报告中会在"综合评分"区加提示：
      "(注意: 仅测试了 1 个场景，得分仅反映该场景参与的指标)"
    - 适合开发过程中快速检查修改效果

  全量测试：
    - 运行 7 个场景，所有指标参与
    - 综合评分有完整参考价值
    - 适合里程碑节点的全面检查
    - 命令: python test_integration.py（不带 --scene）


七、角色设定约定（测试需与这些约定对齐）
----------------------------------------------

  - 名称统一为"爱蜜莉雅"（非"艾米莉娅"）
  - 嫉妒魔女统一为"莎缇拉"（非"莎提拉"）
  - 自称统一为"我"（中文第一人称，不使用日语自称）
  - 帕克状态：前契约者，已休眠（圣域篇后）
  - 证据标注体系：[verbatim]（逐字原作台词）、[impression]（角色精神推断）
  - prompt.md 中各章节标题已更新为第一人称风格（"我是谁""性格与表达"等）


八、如何修改评测策略
----------------------------------------------

  所有核心配置在 tests/questions/config.json：

  {
    "scene_weights": {        ← 改场景权重
      "日常闲聊": 0.10,
      ...
    },
    "metric_weights": {       ← 改指标权重
      "keyword_hit": 0.30,
      ...
    }
  }

  修改后重跑测试即可，不需要改代码。


九、如何新增测试问题
----------------------------------------------

  1. 找到对应的 scene_xxx.json
  2. 在 "cases" 数组中添加一个新对象：
     {
       "id": "daily_006",
       "title": "闲聊话题六",
       "metrics_focus": ["keyword_hit", "character_consistency"],
       "expected_keywords": ["关键词1", "关键词2"],
       "messages": [
         {"role": "user", "content": "你的问题？"}
       ]
     }
  3. 保存，重跑测试

  关键字段说明：
  - metrics_focus: 这个用例要测哪些指标
  - expected_keywords: 期望 AI 回复中出现的关键词列表
  - messages: 对话消息，可以有多轮（第 2 条 user 消息会自动
    带上之前的历史）


十、团队协作建议
----------------------------------------------

  - 克隆项目后双击 setup.bat 完成环境配置，然后双击 run_gui.bat 验证
  - 添加新场景时，在 questions/ 下新建 scene_xxx.json
    （文件名必须以 scene_ 开头）
  - 修改了角色设定文件（prompt.md 等）后，建议至少跑
    daily + core 两个场景验证
  - 报告文件（results/）已加入 .gitignore，不需要提交到仓库
