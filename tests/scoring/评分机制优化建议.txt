一、当前机制回顾
----------------------------------------------
关键词命中率 = 命中数 / 期望关键词总数

  - 逐一检查每个 expected_keywords 是否完整出现在回复中
  - 精确字符串匹配（Python 的 in 运算符）
  - 每命中一个即累积 1/n 分
  - 得分范围 0~1，最终 × 30 换算为满分 30 的整数分

实际测试暴露的问题:
  daily_002 期望 ["甜","好","食"] → 回复中只命中了"甜""好"(2/3)
  原因: AI 用了"甜点""好吃的"等自然表达，单字"食"未单独出现

  daily_003 期望 ["天气","散步","出"] → 回复中只命中了"天气""散步"(2/3)
  原因: "出"作为单字关键词过于苛刻


二、核心问题分析
----------------------------------------------

[问题1] 单字关键词过于简短
  - "好"可能匹配到"你好"而非"好吃"
  - "出"可能是"出发""出现"而非"出门"
  - "会"可能匹配到"社会"而非"一定会"
  - 单字匹配区分度低，测评意义不大

[问题2] 精确匹配过于严格
  - AI 说"甜美的味道"匹配不到期望词"美味"
  - 正常语义等价被当作不匹配
  - 分低了不是因为角色扮演差，而是表达方式自然多变

[问题3] 期望词列表无优先级区分
  - "王选"和"好"对角色测评的重要性完全不同
  - 当前所有关键词权重相同，评分粒度不足


三、优化方案
----------------------------------------------

  ┌──────────────────────────────────────────────────────┐
  │ 方案 A: 分级关键词（推荐优先采用）                    │
  ├──────────────────────────────────────────────────────┤
  │ JSON 格式变更:                                       │
  │ {                                                    │
  │   "required_keywords": ["爱蜜莉雅", "王选"],          │
  │   "bonus_keywords":    ["候选人", "半精灵"]           │
  │ }                                                    │
  │                                                      │
  │ 得分 = required 得分×0.7 + bonus 得分×0.3             │
  │   - required 未命中 → 用例整体降一档                  │
  │   - bonus 命中越多分越高，不命中不惩罚                │
  │                                                      │
  │ 优点: 区分核心考点和表达丰富度                        │
  │ 改动: JSON 结构 + metrics.py                          │
  └──────────────────────────────────────────────────────┘

  ┌──────────────────────────────────────────────────────┐
  │ 方案 B: 模糊匹配                                     │
  ├──────────────────────────────────────────────────────┤
  │ 仅修改 calc_keyword_hit()，不改 JSON:                 │
  │  1. 最小匹配长度 >= 2 个字符 → 自动跳过单字词         │
  │  2. 子串包含匹配: "甜美的味道" 匹配关键词 "甜味"      │
  │                                                      │
  │ 优点: 改动最小                                       │
  │ 缺点: 过度放宽可能误判（"国家"匹配"我家"）            │
  └──────────────────────────────────────────────────────┘

  ┌──────────────────────────────────────────────────────┐
  │ 方案 C: 惩罚扣分制                                   │
  ├──────────────────────────────────────────────────────┤
  │ score = 1.0 - (缺少的关键词数 × 扣分系数)             │
  │                                                      │
  │ 与当前数学等价，但心理模型更直观:                     │
  │ "每少一个扣 N 分" vs "命中 M/N 得 M/N"                │
  │                                                      │
  │ 优点: 团队理解更直观，无技术成本                       │
  └──────────────────────────────────────────────────────┘


四、Quick Fix 清单（无需改代码）
----------------------------------------------
直接修改 JSON 问题集中不合理的关键词:

  文件                  问题词       建议替换
  ──────────────────────────────────────────────────
  scene_daily.json      "好","食"    "甜食"/"点心"/"蛋糕"
  scene_daily.json      "出"         "出门"/"外面"/"散步"

  改完重跑即可看到关键词命中率提升。


五、推荐实施路径
----------------------------------------------
  第一阶段（立即）
    → Quick Fix: 修改 JSON 中不合理的单字/短词
    → 重跑全量测试，对比前后分数变化

  第二阶段（短期）
    → 实施方案 A: 在 1 个 JSON 文件中试点分级关键词
    → 测试通过后推广到全部场景

  第三阶段（中期）
    → 可选实施 B: 引入子串容错匹配
    → 结合 A+B 形成最终版评测体系


六、新旧对比（以 daily_002 为例）
----------------------------------------------
  当前: expected_keywords = ["甜","好","食"]
        → AI 得 2/3 = 0.67

  方案 A: required = ["甜"], bonus = ["蛋糕","甜食"]
          → AI 必得 0.7+0.3 = 1.0

  方案 B: 子串匹配 + 最小长度>=2
          → 自动跳过"好""食"（长度<2），仅评估"甜"= 1.0

  Quick Fix: expected_keywords = ["甜","甜点","蛋糕"]
            → AI 得 3/3 = 1.0


七、角色设定变更对测试的影响
----------------------------------------------
  第一轮中文语境优化（2026-06-16，v0.1 → v0.2）后的注意事项:

  1. 自称变更
     - 旧: "私"或"我"（含日语第一人称）
     - 新: 统一为"我"（纯中文第一人称）
     - 影响: conftest.py 中的 CHARACTER_PERSON_PATTERNS 需更新

  2. 证据标注变更
     - 旧: [artifact]（官方资料）+ [verbatim]（逐字台词）
     - 新: [verbatim]（逐字原作台词）+ [impression]（角色精神推断）
     - 影响: test_basic.py 中的证据覆盖检测已同步更新

  3. 章节结构变更
     - 旧: prompt.md 分"核心设定""身份与背景""角色呈现指南"等
     - 新: prompt.md 分"我是谁""性格与表达""说话风格指引"等
     - 影响: test_basic.py 中的章节完整性检测已同步更新

  4. 交互文件重定位
     - 旧: interaction.md 包含说话语气 + 互动风格
     - 新: interaction.md 专注互动场景指南（说话方式移至 speaking.md）
     - 影响: 测试用例中的期望文本可能需调整
