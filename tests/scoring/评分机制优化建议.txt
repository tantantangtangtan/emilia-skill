一、当前机制回顾
----------------------------------------------
关键词命中率 = 命中数 / 期望关键词总数

  - 逐一检查每个 expected_keywords 是否完整出现在回复中
  - 精确字符串匹配（indexOf），区分大小写
  - 每命中一个即累积 1/n 分

实际测试暴露的问题:
  daily_002 期望 ["甜","好","食"] → 回复中只命中了"甜"(1/3=0.333)
  原因: AI 用了"甜点""甜美的"等自然表达，未使用"食"和单独"好"字

二、核心问题分析
----------------------------------------------

[问题1] 单字关键词"好""食""出""会"过于简短
  风险:
    - 如"好"可能匹配到"你好"而非"好吃"
    - 如"出"可能是"出发""出现"而非"出门"
    - 单字匹配区分度低，测评意义不大
  影响: 会导致不应该低分的优质回答被误判

[问题2] 精确匹配过于严格
  风险:
    - AI 说"甜美的味道"匹配不到期望词"美味"
    - AI 说"我想要成为王"匹配不到期望词"会成为王"
    - 正常语义等价被当作不匹配
  影响: 分低了不是因为角色扮演差，而是表达方式自然多变

[问题3] 期望词列表缺乏"必需/可选"区分
  风险:
    - 全部关键词权重相同，无法区分核心词和锦上添花词
    - "王选"和"好"对角色测评的重要性完全不同
  影响: 评分粒度不足

三、优化方案（3 个独立方案，可单独或组合使用）
----------------------------------------------

┌─────────────────────────────────────────────────────────┐
│  方案 A: 分级关键词（推荐优先采用）                       │
├─────────────────────────────────────────────────────────┤
│  在 JSON 问题集中将 expected_keywords 拆分为两级:        │
│                                                         │
│  JSON 格式变更:                                         │
│  {                                                      │
│    "required_keywords": ["爱蜜莉雅", "王选"],  // 必须   │
│    "bonus_keywords":    ["候选人", "半精灵"]   // 加分   │
│  }                                                      │
│                                                         │
│  计分方式:                                              │
│    总分 = required 部分得分 × 0.7 + bonus 部分得分 × 0.3 │
│    - 任一 required 未命中 → 该用例整体降一档             │
│    - bonus 命中越多分越高，但不命中不惩罚                │
│                                                         │
│  优点: 区分核心考点和表达丰富度                           │
│  缺点: 需修改 JSON 结构 + metrics.py                     │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  方案 B: 模糊匹配（子串/语义容错）                        │
├─────────────────────────────────────────────────────────┤
│  仅修改 calc_keyword_hit()，不改 JSON:                   │
│                                                         │
│  变更内容:                                              │
│  1. 最小匹配长度 >= 2 个字符 → 单字词废弃                │
│  2. 子串包含匹配: "甜美的味道" 匹配关键词 "甜味"         │
│  3. 可选: 加入同义扩展表，如 "甜" → ["甜","甘","蜜"]     │
│                                                         │
│  计分方式:                                              │
│    - 保持 命中数/总数 不变                               │
│    - 仅放宽"命中"的判定条件                              │
│                                                         │
│  优点: 改动最小（仅 metrics.py）                         │
│  缺点: 过度放宽可能导致误判（如"国家"匹配"我家"）        │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  方案 C: 惩罚扣分制（反向计分）                           │
├─────────────────────────────────────────────────────────┤
│  满分起算，缺一个扣一定比例:                              │
│                                                         │
│  score = 1.0 - (缺少的关键词数 × 扣分系数)               │
│  其中扣分系数 = 1.0 / 总关键词数                         │
│                                                         │
│  本质与当前相同（数学等价），但心理模型更直观:             │
│  "每少一个扣 0.33 分" vs "命中 1/3 得 0.33"             │
│                                                         │
│  优点: 无技术收益，但团队理解更直观                       │
│  缺点: 无                                               │
└─────────────────────────────────────────────────────────┘
